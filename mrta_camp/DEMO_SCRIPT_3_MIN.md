# 3-Minute Demo Script

## 0:00-0:25 Problem
Bangkok already has a large rail network, but many people still hesitate before using it. They are unsure where to transfer, how much it will cost, whether the 20-baht scheme applies, and what the station experience looks like.

## 0:25-0:55 Map and MRTA Mode
Open the app and show the 3D network. Then click `เพิ่มเติม → Demo Mode → เริ่มเดโม`.

Say: this instantly focuses the demo on the MRTA context: Blue, Purple, Yellow, and Pink lines, while the full network remains available.

## 0:55-1:35 Route + Fare
Show the route card. Explain that the app gives line-by-line instructions, transfer walking hints, time estimate, normal fare, and 20-baht comparison.

Click `บริการ → เทียบราคา 20฿ vs ปกติ`.

Say: this is not a fake promotion screen. It uses the same route calculation as the planner and labels whether fare data is exact from open data or estimated.

## 1:35-2:10 First Ride Simulator
Click `บริการ → ซ้อมนั่งครั้งแรก`.

Say: this is the core user-experience innovation. The app does not only tell users where to go; it trains a first-time rider before they leave home.

## 2:10-2:35 Data Trust
Click `เพิ่มเติม → ที่มาข้อมูลแบบละเอียด` and `สถานะข้อมูล`.

Say: the app separates official/open data, open map data, estimated data, and local-only data. This prevents stale promotions or fake realtime information.

## 2:35-3:00 Close
Say: Bangkok Transit 3D is a scalable passenger-confidence layer for rail adoption. The next production step is official realtime incident feed, official accessibility data, and operator API integration.
