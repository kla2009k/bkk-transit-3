## สไลด์ 1 — ชื่อผลงาน

**"MRT Digital Twin" — แผนที่ 3 มิติมีชีวิตของรถไฟฟ้ามหานคร**
วางแผนเดินทาง เห็นราคาก่อนออกจากบ้าน ซ้อมนั่งรถไฟฟ้าได้เหมือนอยู่ที่สถานีจริง

*(แถวรอง เล็กๆ ใต้ชื่อ)* ต้นแบบใช้งานได้จริงแล้ววันนี้: ครบ 193 สถานี ทุกสายในกรุงเทพฯ จากข้อมูลเปิดภาครัฐ + OpenStreetMap

## สไลด์ 2 — ที่มาและแนวคิดของผลงาน

พวกเราสังเกตว่าคนรอบตัวจำนวนมาก "ไม่กล้า" ใช้รถไฟฟ้า — ผู้ใหญ่ที่ไม่เคยนั่ง กลัวงงตอนเปลี่ยนสาย ญาติต่างจังหวัดไม่รู้ว่าค่าโดยสารจะโดนเท่าไหร่ นักท่องเที่ยวเปิดแผนที่ 2 มิติแล้วก็ยังหลงในสถานี ทั้งที่โครงข่ายรถไฟฟ้าของเราขยายไปถึง 10 สาย เกือบ 200 สถานีแล้ว — **ระบบรางไทยไปไกลกว่าความมั่นใจของผู้ใช้**

[สมาชิกทีมเล่าประสบการณ์จริง 1 เรื่อง เช่น พาคุณยายไปหาหมอด้วย MRT แล้วหลงที่จุดเปลี่ยนสาย — ใส่ 2-3 ประโยค เรื่องจริงจำง่ายกว่าสถิติ]

แรงบันดาลใจของเรา: เกม/แผนที่ 3 มิติอย่าง Google Earth ทำให้คน "รู้จักที่ที่ไม่เคยไป" ได้ — ถ้าระบบรถไฟฟ้าทั้งกรุงเทพฯ มีเวอร์ชัน 3 มิติที่มีชีวิต (รถวิ่งจริงตามตารางเวลา เมืองจริง ตึกจริง ราคาจริง) ทุกคนก็ "ซ้อมเดินทาง" ได้ก่อนออกจากบ้าน ความกลัวจะหายไป และคนจะหันมาใช้รถไฟฟ้ามากขึ้น

## สไลด์ 3 — วัตถุประสงค์ (ผลงานนี้ใช้แก้ปัญหาอะไร)

ปัญหาที่แก้ (จากการสัมภาษณ์ผู้ใช้ [XX] คน ช่วง [X-XX ก.ค. 2569]):
1. **ไม่รู้ราคาก่อนเดินทาง** — [XX]% ของผู้ตอบไม่แน่ใจว่าทริปข้ามสาย (เช่น MRT→BTS) จะเสียเท่าไหร่ และไม่รู้เงื่อนไขนโยบาย 20 บาทตลอดสาย (เปลี่ยนสายใน 30 นาที / จบทริปใน 180 นาที)
2. **วางแผนเส้นทางข้ามสายยาก** — แอปแผนที่ทั่วไปแสดงเส้น 2 มิติ ไม่เห็นว่าจุดเปลี่ยนสายจริงๆ ต้องเดินแค่ไหน สถานีอยู่บนดินหรือใต้ดิน
3. **ผู้ใช้หน้าใหม่ขาดความมั่นใจ** — ผู้สูงอายุ/ผู้มาเยือน ไม่มีที่ให้ "ลองก่อน" — ต้องไปงงหน้างานจริง

วัตถุประสงค์:
- ให้ทุกคนวางแผนทริปรถไฟฟ้า **เห็นเส้นทาง + เวลา + ราคาจริงทั้ง 2 แบบ (มี/ไม่มีสิทธิ 20฿) ใน 10 วินาที**
- ลดกำแพงผู้ใช้หน้าใหม่ด้วยการ "ซ้อมเดินทางเสมือน" (นั่งมุมหัวขบวน เห็นเมืองจริง)
- เพิ่มสัดส่วนการใช้ขนส่งสาธารณะ → ลดรถยนต์ส่วนตัว ลด PM2.5 และ CO₂

## สไลด์ 5 — วิธีการดำเนินงาน

**สถานะปัจจุบัน: ต้นแบบใช้งานได้จริงแล้ววันนี้ บนเบราว์เซอร์ทุกเครื่อง ไม่ต้องติดตั้งแอป (ติดตั้งเป็น PWA ได้)**

ขั้นตอนที่ทำแล้ว (มิ.ย.-ก.ค. 2569):
1. ดึงข้อมูลจริงจากแหล่งเปิด: สถานี+ราง 10 สาย 193 สถานี + อาคาร ~111,000 หลัง + ทางออกสถานี 432 จุด (OpenStreetMap), ตารางค่าโดยสารทางการ 6,953 คู่สถานี + ผู้โดยสารรายเดือน (Open Data กรมการขนส่งทางราง)
2. เอนจิน 3 มิติบนเว็บ (Three.js): เมืองจริงตามสเกล รถไฟวิ่งตามความถี่จริงต่อช่วงเวลา
3. วางแผนเส้นทางด้วยภาษาเดียวกับป้ายในสถานี: รหัสสถานี (BL13, N8) + "ขึ้นขบวนมุ่งหน้า..." + ทางออกปลายทาง (♿ ระบุจุดที่รองรับวีลแชร์) + ราคา 2 โหมด (20฿ ตามเงื่อนไข / ราคาปกติจากตารางทางการ)
4. **"ซ้อมเดินทางแบบ 3D"** — กล้องพาบินตามเส้นทางจริงต้นทาง→ปลายทาง หยุดอธิบายทุกจุดเปลี่ยนสาย + CO₂ ที่ประหยัดต่อทริป + โหมดตัวหนังสือใหญ่สำหรับผู้สูงอายุ

**แผนต่อยอดในบริบท รฟม. (1-3 ปี):**
- **ปีที่ 1** — ทดลองนำร่อง: embed ในเว็บ/จอ interactive ในสถานี MRT · โหมดเน้นสาย รฟม. (มีแล้ว: `?mrta`) · เก็บ insight เส้นทางที่ถูกค้นมากสุด · ปรับตามผลสัมภาษณ์ผู้ใช้จริง
- **ปีที่ 2** — เชื่อมข้อมูลภายใน: ตารางเดินรถ/ดีเลย์เรลไทม์ของ รฟม. · **สายสีส้มก่อนเปิดบริการ — ให้ประชาชน "เห็นและซ้อมนั่ง" เส้นทางใหม่ล่วงหน้า** (เครื่องมือ PR ที่จับต้องได้)
- **ปีที่ 3** — digital twin เต็มรูป: ความหนาแน่นเรียลไทม์, in-station navigation, ขยายสู่ระบบ feeder

## สไลด์ 6 — การนำไปใช้ประโยชน์ (แพร่หลายแค่ไหน)

**ใช้ได้แพร่หลายทันที ไม่จำกัดวง:**
- เป็นเว็บ — เปิดได้ทุกอุปกรณ์ ไม่ต้องลงแอป ไม่ต้องสมัครสมาชิก ต้นทุนผู้ใช้ = 0
- ข้อมูลทั้งหมดมาจากแหล่งเปิด (OSM + Open Data ภาครัฐ) → ไม่ติดลิขสิทธิ์ ขยายไปเมืองอื่นได้ (ภูเก็ต เชียงใหม่ ขอนแก่น ที่มีแผนระบบราง)
- **สำหรับ รฟม. โดยตรง:**
  - embed ในเว็บ/แอปทางการ เป็นแผนที่โต้ตอบให้ผู้โดยสาร
  - ใช้เป็นสื่อประชาสัมพันธ์เส้นทางเปิดใหม่ (ม่วงใต้, ส้ม) — ให้ประชาชน "เห็น" เส้นทางก่อนเปิดจริง
  - จอ interactive ในสถานี/งานนิทรรศการ
  - ฐานสู่ digital twin เต็มรูปแบบขององค์กร (ต่อ API เรลไทม์ในอนาคต)
- สถานศึกษาใช้สอนเรื่องผังเมือง/ขนส่งสาธารณะได้ฟรี

## สไลด์ 8 — งบประมาณในการผลิตผลงาน

**จุดแข็ง: ต้นทุนต่ำมากเพราะเป็นเว็บ static + ข้อมูลเปิด**

| รายการ | ต้นทุน |
|---|---|
| พัฒนาต้นแบบ (ทำเสร็จแล้ว — แรงงานทีม + เครื่องมือโอเพนซอร์ส) | 0 บาท |
| Hosting เว็บ (Netlify/Cloudflare free tier รองรับหลักหมื่น visit/เดือน) | 0 บาท/เดือน |
| โดเมน .app หรือ .co | ~400 บาท/ปี |
| ข้อมูล (OSM + Open Data กรมการขนส่งทางราง) | 0 บาท |
| **รวมต้นทุนให้บริการ** | **~400 บาท/ปี** |
| กรณีขยาย (ผู้ใช้ >100k/เดือน + อัพเดตข้อมูลอัตโนมัติ): cloud hosting | ~300-1,000 บาท/เดือน |

ต้นทุนต่อการใช้งาน 1 ครั้ง ≈ **ต่ำกว่า 0.01 บาท** — เทียบไม่ได้กับสื่อสิ่งพิมพ์/ป้ายประชาสัมพันธ์

## สไลด์ 9 — ผลที่คาดว่าจะได้รับ

**ต่อผู้คน:** ผู้ใช้หน้าใหม่มีความมั่นใจ วางแผนได้ใน 10 วินาที รู้ราคาก่อนเดินทาง ไม่มี "บิลช็อก" — เป้าหมาย: ผู้ใช้ [1,000+] คนใน 3 เดือนแรกหลังเปิดตัว [วัดจาก analytics]

**ต่อสังคม:** ลดกำแพงการเข้าถึงขนส่งสาธารณะของกลุ่มเปราะบาง — ทำแล้วในต้นแบบ: โหมดตัวหนังสือใหญ่สำหรับผู้สูงอายุ + ♿ ระบุทางออกที่รองรับวีลแชร์รายสถานี + UI สองภาษา; ทุกทริปที่ย้ายจากรถยนต์มา MRT ระยะ 10 กม. ลด CO₂ ~1.2-1.7 กก. **(แอปแสดงตัวเลขนี้ต่อทริปแล้ววันนี้ พร้อมเทียบเป็น "วันที่ต้นไม้ดูดซับ")**

**ต่อสิ่งแวดล้อม:** ระบบรางกรุงเทพฯ มีผู้โดยสาร >40 ล้านเที่ยว/เดือน (Open Data กรมการขนส่งทางราง มี.ค. 2569) — ถ้าเครื่องมือนี้ช่วยดึงคนจากรถยนต์เพิ่มเพียง **0.1% = ~40,000 เที่ยว/เดือน ≈ ลด CO₂ 50-70 ตัน/เดือน (~600-840 ตัน/ปี)** (ส่วนต่าง ~125 gCO₂e/ผู้โดยสาร-กม.) — **เทียบเท่า ~7% ของที่โครงการ Solar Rooftop 17 MWp ของ รฟม. ทั้งโครงการลดได้ (11,900 ตัน/ปี) โดยไม่ต้องติดตั้งฮาร์ดแวร์ใดๆ เพิ่ม** และสอดคล้องเป้าหมายประเทศ Carbon Neutrality 2050; ในแอปมี **Green Journey Ledger** สะสม CO₂ ที่ผู้ใช้แต่ละคนช่วยลด (เทียบทั้งรถยนต์และมอเตอร์ไซค์) + สอดแทรกความรู้พลังงานสะอาดของ รฟม. (solar rooftop, regenerative braking) ในโหมดสอนนั่งครั้งแรก — เปลี่ยนผู้โดยสารให้เป็นผู้เล่าเรื่องความยั่งยืนขององค์กรต่อ

**ต่อ รฟม. (โอกาสทางธุรกิจ):** ช่องทางสื่อสารบริการที่คนเข้าใจใน 10 วินาที · **เครื่องมือ PR สายสีส้มก่อนเปิด — ให้ประชาชนซ้อมนั่งเส้นทางใหม่ล่วงหน้าแบบ 3D** · insight เส้นทางที่ถูกค้นมากสุดเพื่อวางแผนบริการ · ฐานสู่ digital twin องค์กรโดยไม่เริ่มจากศูนย์ · ต้นทุนให้บริการ ~400฿/ปี

**แผนพัฒนาต่อ (Idea Development Plan) — สอดคล้อง slide 5:**
ปีที่ 1: นำร่อง embed เว็บ/จอในสถานี + user research ต่อเนื่อง → ปีที่ 2: API เรลไทม์ รฟม. + สายสีส้ม pre-launch → ปีที่ 3: digital twin เต็มรูป (ความหนาแน่นเรียลไทม์, in-station navigation, ขยาย feeder/เมืองอื่น)
